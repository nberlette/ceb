Class AbstractBusBuilder<E>

The builder handles the integration of the Event/Message Driven approach provided natively by <ceb/>. Its purpose is to provide a quick and efficient way to interact with the Bus within the Custom Element. Therefore, it is easy to execute queries to get data, execute commands to generate side effects and finally listen to events to react on side effects.

First of all, the Bus is created and set to a readonly property, by default its name is bus. By default, the global channel is window, it can be overridden with {@link AbstractBusBuilder.global}

Then, subscriptions can be registered with AbstractBusBuilder.subscribe. When the builder is used as a decorator, then AbstractBusBuilder.subscribe provides an instance of BusSubscriptionBuilder which can be used to easily configure a subscription.

The bus starts when the Custom Element is connected, c.f. connectedCallback and, stops when it is disconnected, c.f. disconnectedCallback.

The builder can be used many times by Custom Element. There will be a unique Bus instance per properties.
